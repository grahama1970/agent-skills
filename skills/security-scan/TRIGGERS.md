# Security-Scan Triggers

The security-scan skill is invoked when the user mentions any of these keywords or phrases:

## Primary Triggers
- security scan
- security-scan
- sast
- sast scan
- static analysis
- code scan

## Vulnerability Triggers
- vulnerability scan
- find vulnerabilities
- check for vulnerabilities
- security audit
- security check

## Dependency Triggers
- dependency audit
- dep audit
- dependency scan
- check dependencies
- vulnerable dependencies
- pip-audit
- trivy

## Secrets Triggers
- secrets detection
- find secrets
- hardcoded secrets
- leaked credentials
- api keys exposed
- gitleaks

## Tool-Specific Triggers
- run semgrep
- run bandit
- run gitleaks
- run trivy

## Example Usage
```
User: "Run a security scan on this project"
User: "Check for vulnerabilities in the codebase"
User: "Find any hardcoded secrets"
User: "Audit the dependencies for CVEs"
```
